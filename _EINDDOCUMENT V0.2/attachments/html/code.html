Code review

HTML semantiek
Op 12 juli hielden wij als groep een code review. Deze review heeft mij veel verder geholpen en tevens gewezen op een aantal "bad practices" die ik toepaste in mijn project. Zo gebruikte ik veel "div's" en weinig HTML-elementen die beter geschikt waren. Het gevolg daarvan is dat browsers minder inzicht hebben in het doel van je HTML-secties. Blinde mensen zullen daardoor bijvoorbeeld meer moeite hebben met het gebruik van de applicatie. Ook heb daarom zoveel mogelijk elementen herschreven.

Gebruik zo min mogelijk jQuery
jQuery, een Javascript library die het schrijven van Javascript makkelijker maakt, heeft veel invloed op de performance van je applicatie. In eerste instantie schreef ik zo'n beetje alles met jQuery. Vervolgens heb ik alle jQuery elementen die ik in vanilla Javascript kon schrijven, herschreven in Javascript. Mijn applicatie werd daardoor aanzienlijk sneller.

Deel CSS in componenten op
Een van de voordelen van SCSS is dat je "partials" kunt gebruiken. Dat zijn opgedeelde stukken code die ervoor zorgen dat jij het overzicht kunt behouden en dat je ze kunt hergebruiken. Nu hoef je voor CSS in principe niks te hergebruiken, maar werk je wel sneller.

Gebruik zo min mogelijk CSS code in Javascript
Voor anderen is het lastig werken als zij erachter moeten komen dat veel CSS-code ergens in je Javascript verstopt zit. Ik had daarom veel dynamische stijlaanpassingen herschreven. Wat eerst het volgende was:

document.getElementsByClassName('class').style.backgroundColor = "#something";

Herschreef ik als:

document.getElementsByClassName('class').classList.add('new-background');

Verander pixels in andere CSS-eenheden
Ik maakte heel veel gebruik van pixels. Het voordeel is dat je op de pixel precies kunt aangeven welk formaat een element moet zijn. Het nadeel is dat het gebruik ervan resulteert in een minder “responsive” applicatie. Waar ik bijvoorbeeld het volgende gebruikte:


				.element {
					margin: 20px 10px;
					line-height: 20px;
				}

Herschreef ik dit als:


				.element {
					margin: 1.1rem 0.55rem;
					line-height: 1.1rem;
				}
