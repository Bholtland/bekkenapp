Progressive Web Apps

Wat is een PWA precies?

Een Progressive Web App is de naam die wordt gegeven aan Web app, die gebruik maakt van een aantal nieuwe technieken en api's. Het doel van de Progressive Web App is om een Web app de zelfde functionaliteiten en karakteristieken te geven als een Native app. Op dit moment zijn de meest belangrijke functies het Manifest, de Service Worker en push/notifications api.

Manifest.json

Het manifest is niets meer dan een JSON bestand waarin bepaalde eigenschappen van de PWA worden aangegeven. Hieronder vallen app iconen, naamgeving, kleur, wrapperspecificaties (fullscreen, met of zonder url etc.

De manifest zorgt er eigenlijk voor dat je een website/webapp in een native jasje kunt steken, zodat het voelt als een native app. Deze functie werkt echter niet zonder een Service Worker.

Service Worker

Een Service Worker is een Javascript bestand dat wordt uitgevoerd om extra functies uit te voeren. De Service Worker is een extra schakel tussen de Server en de Client. Een Service Worker biedt de mogelijkheid om bestanden te kunnen cachen (zodat de web app ook offline kan werken). Daarnaast is de Service Worker altijd bereikbaar. Dit is vooral handig voor bijvoorbeeld Push notificaties. De Service Worker bevindt zich in een ruststand. Zodra de push server contact maakt, ontwaakt de Service Worker en voert deze taken uit; Ook als de browser, en zelfs de telefoon, niet actief worden gebruikt. Een Service Worker werkt alleen over een https verbinding.

API's

De Push en Notifications api's zijn voorbeelden van api's die gebruikt kunnen worden om functies aan te bieden die vergelijkbaar zijn met native apps. Er verschijnen steeds meer api's die extra mogelijkheden geven. Deze api's zorgen ervoor dat we de batterijstatus kunnen aflezen, de lichtintensiteit kunnen meten, gebruik kunnen maken van de afstandssensor, de gyroscoop en de accelerometer.
